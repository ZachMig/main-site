This is a demonstration of a REST server connected to a database, and this client webpage to use the service.
The REST service is a Spring Boot project written in Java.
The data is contained in a MySQL database which is hooked up to the REST service via Hibernate ORM.
The REST service and database are both hosted on an AWS EC2 instance.
Routing for zachmig.com and api.zachmig.com are handled by the AWS Route 53 service.
This page is contained in an AWS S3 bucket. Requests to www.zachmig.com are passed to this bucket as well.

This page represents the client side.
There isn't anything on the front end in the way of exception handling or compatibility.
The Javascript waits for the user to select a country from the dropdown and uses the input to build a URL for a fetch() call to www.api.zachmig.com.
The JSON response is then loaded into a table and displayed on the page.

The fetch() call is made to "http://www.api.zachmig.com:8080/country?name=United_States_of_America".
This URL points to an endpoint of a REST API.
Upon receiving the GET request, the REST service builds a SELECT query using the country name and runs it against the database.
The results of the query are serialized into objects and then finally into a JSON response which is served to the requester.
